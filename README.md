# 📊 Análise de Indicadores de Qualidade Automotiva | Dashboard Interativo em Excel
Principais resultados:

- 1.406 ocorrências registradas em 2025
- Taxa de defeitos: 3,99%
- Área mais crítica: Montagem
- Modelo mais impactado: SUV C
- Principal problema: Torque fora da especificação
- Aplicação de Pareto, Correlação, Ishikawa, 5 Porquês e 5W2H

  ## 📌 Principais Insights

- O problema "Torque fora da especificação" apresentou a maior concentração de ocorrências no período analisado.
- A área de Montagem concentrou o maior número de registros de defeitos.
- A análise de dispersão indicou correlação positiva moderada entre produção e defeitos.
- Observou-se forte correlação entre defeitos e ocorrências de pós-venda.
- As análises de Ishikawa e 5 Porquês permitiram identificar causas-raiz relacionadas à padronização dos processos e validação das operações.

Este projeto apresenta o desenvolvimento de um dashboard interativo em Microsoft Excel para análise de indicadores de qualidade automotiva referentes ao ano de 2025. A solução foi desenvolvida utilizando uma base de dados fictícia criada exclusivamente para fins de estudo e composição de portfólio, simulando um cenário real de monitoramento da qualidade em uma indústria automotiva.O dashboard foi projetado para apoiar a tomada de decisão por meio da consolidação de indicadores, identificação de áreas críticas, análise dos principais problemas de qualidade e acompanhamento das ocorrências ao longo do período analisado.

## 🎯 Objetivo
Desenvolver um dashboard interativo capaz de transformar dados operacionais em informações estratégicas, permitindo identificar os principais indicadores de qualidade registrados durante o ano de 2025. O projeto busca demonstrar a aplicação de técnicas de análise de dados utilizando Microsoft Excel para apoiar a tomada de decisão baseada em dados.
---
## 🛠 Ferramentas Utilizadas
- **Microsoft Excel**
  - Tabelas Dinâmicas
  - Gráficos Dinâmicos
  - Segmentação de Dados (Slicers)
  - KPIs
  - Formatação Condicional
  - Fórmulas
  - Diagrama de Pareto

- **Git**
  - Controle de versão do projeto

- **GitHub**
  - Publicação e documentação do projeto

## 📈 Indicadores Monitorados

- Total de Ocorrências
- Taxa de Defeitos (%)
- Modelo Crítico
- Área Crítica
- Problema Principal
- Evolução Mensal das Ocorrências
- Distribuição de Ocorrências por Área
- Correlação entre Defeitos e Pós-Venda
- Correlação entre Produção e Defeitos

## 🔎 Metodologia Utilizada

1. Construção da base de dados fictícia
2. Tratamento e padronização dos dados
3. Criação das tabelas dinâmicas
4. Desenvolvimento dos KPIs
5. Construção dos gráficos e segmentações
6. Aplicação do Diagrama de Pareto
7. Análise de Correlação por Dispersão
8. Identificação das causas raízes utilizando:
   - Ishikawa
   - 5 Porquês
9. Elaboração dos planos de ação utilizando 5W2H
10. Consolidação dos insights para tomada de decisão

## 💡 Competências Demonstradas

Durante o desenvolvimento deste projeto foram aplicadas competências relacionadas a:

- Organização e tratamento de dados
- Criação de dashboards interativos
- Desenvolvimento de indicadores (KPIs)
- Análise exploratória de dados (EDA)
- Visualização de dados
- Comunicação de informações para apoio à tomada de decisão
- Documentação técnica de projetos

## ⚠️ Limitações

Os dados utilizados neste projeto são fictícios e foram desenvolvidos exclusivamente para fins de estudo e composição de portfólio. Apesar disso, foram estruturados para representar um cenário plausível da indústria automotiva e permitir a aplicação de técnicas de análise de dados e ferramentas da qualidade.
